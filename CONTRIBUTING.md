# Contributing to Kiara

Thank you for your interest in contributing to Kiara.

## Repository access

- **Read:** Anyone can read and clone this repository.
- **Fork:** Anyone can fork the repository to their own account and work on their fork.
- **Push to this repo:** Only authorized maintainers can push directly to the default branch (`main`). This is enforced by branch protection rules.

If you are not an authorized maintainer, please contribute by **forking the repo and opening a pull request** (see below). Do not request push access to this repository.

## How to contribute (fork and pull request)

1. **Fork** this repository to your GitHub account (click "Fork" on the repo page).
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/kiara-slm-project.git
   cd kiara-slm-project
   ```
3. **Create a branch** for your change:
   ```bash
   git checkout -b my-feature
   ```
4. **Make your changes**, commit, and push to your fork:
   ```bash
   git add .
   git commit -m "Description of your change"
   git push origin my-feature
   ```
5. **Open a pull request** from your fork’s branch to this repo’s `main` branch on GitHub. An authorized maintainer will review and merge if appropriate.

## What you can contribute

- Bug fixes and feature improvements
- Documentation updates
- Bug reports and feature requests (via GitHub Issues)
- Code reviews and feedback on pull requests

## Code and conduct

- Follow existing code style (see `pyproject.toml` and project formatting tools).
- Keep changes focused; one logical change per pull request when possible.
- Be respectful and constructive in discussions.

Thank you for contributing.
